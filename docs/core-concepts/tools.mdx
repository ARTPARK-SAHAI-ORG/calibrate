Tools enable your agents to produce structured outputs or call external APIs. They define the parameters and format for extracting information from conversations.

## Tool Types

Calibrate supports two types of tools:

### Structured Output (Default)

Use `"type": "structured_output"` (or `"client"`) for tools that extract structured data from conversations. The LLM calls the tool with the specified parameters, and Calibrate returns a status acknowledgment.

```json
{
  "type": "structured_output",
  "name": "plan_next_question",
  "description": "Plan the next question to ask the user",
  "parameters": [
    {
      "id": "next_unanswered_question_index",
      "type": "integer",
      "description": "Index of the next question to ask",
      "required": true
    },
    {
      "id": "questions_answered",
      "type": "array",
      "description": "List of question indices that have been answered",
      "items": {"type": "integer"},
      "required": true
    }
  ]
}
```

### Webhook

Use `"type": "webhook"` for tools that call external APIs. When the LLM invokes a webhook tool, Calibrate makes an actual HTTP request to the specified endpoint and returns the response to the LLM.

```json
{
  "type": "webhook",
  "name": "submit_form",
  "description": "Submit collected form data to the backend API",
  "parameters": [],
  "webhook": {
    "method": "POST",
    "url": "https://api.example.com/submit",
    "timeout": 20,
    "headers": [
      {"name": "Authorization", "value": "Bearer YOUR_API_TOKEN"},
      {"name": "Content-Type", "value": "application/json"}
    ],
    "queryParameters": [
      {
        "id": "api_key",
        "type": "string",
        "description": "API key for authentication",
        "required": true
      }
    ],
    "body": {
      "description": "Form data to submit",
      "parameters": [
        {
          "id": "patient_name",
          "type": "string",
          "description": "Patient's full name",
          "required": true
        },
        {
          "id": "patient_age",
          "type": "integer",
          "description": "Patient's age",
          "required": false
        }
      ]
    }
  }
}
```

**Required webhook fields:**
- `url` - The endpoint URL
- `method` - HTTP method (GET, POST, PUT, PATCH, DELETE)
- `headers` - Array of header objects (can be empty `[]`)

**Optional webhook fields:**
- `timeout` - Request timeout in seconds (default: 20)
- `queryParameters` - URL query parameters
- `body` - Request body parameters (for POST, PUT, PATCH)

## Webhook Behavior by Module

| Module | Structured Output | Webhook |
|--------|-------------------|---------|
| **LLM Tests** | Returns `{"status": "received"}` | Logs details only (no HTTP call) |
| **Text Simulations** | Returns `{"status": "received"}` | Makes actual HTTP call |
| **Voice Simulations** | Returns `{"status": "received"}` | Makes actual HTTP call |
| **Interactive Testing** | Returns `{"status": "received"}` | Makes actual HTTP call |

## Inbuilt Tools

The following tool is built-in and enabled by default:

| Tool | Description |
|------|-------------|
| **end_call** | Allows the agent to end the conversation |

## Parameter Types

Tools support the following parameter types:

| Type | Description | Example |
|------|-------------|---------|
| `string` | Text value | `"hello"` |
| `integer` | Whole number | `42` |
| `number` | Decimal number | `3.14` |
| `boolean` | True/false | `true` |
| `array` | List of values | `[1, 2, 3]` |
| `object` | Nested object | `{"key": "value"}` |

For arrays, specify the item type with `"items"`:

```json
{
  "id": "tags",
  "type": "array",
  "description": "List of tags",
  "items": {"type": "string"},
  "required": false
}
```

For enums, specify allowed values with `"enum"`:

```json
{
  "id": "priority",
  "type": "string",
  "description": "Task priority",
  "enum": ["low", "medium", "high"],
  "required": true
}
```
